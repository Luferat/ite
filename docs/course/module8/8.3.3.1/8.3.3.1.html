Conectividade de Rede e Email

Configurando o Email

Softwares de email podem ser instalados como parte de um navegador web ou como um aplicativo independente. Qualquer programa de email pode ser usado no Windows 7. O Windows Live Mail é um programa de email recomendado pela Microsoft. Ele gerencia várias contas de email, calendários e contatos, como mostrado na Figura 1. Para instalar o Windows Live Mail, baixe e instale o Windows Essentials da Microsoft. O Windows Live Mail está incluso no Windows Essentials.

Você deve ter as seguintes informações ao configurar uma conta de email:

  • Nome de Exibição - Este pode ser o seu verdadeiro nome, apelido ou qualquer outro nome que você quer que as pessoas vejam.

  • Endereço de email - Este é o endereço que as pessoas utilizam para enviar emails para você. Um endereço de email é um nome de usuário seguido do símbolo @ e o domínio do servidor de email (user@example.net).

  • Protocolo usado pelo servidor de emails - Diferentes protocolos fornecem diversos serviços de email.

  • Nomes de servidores de email de entrada e saída - Esses nomes são fornecidos pelo administrador da rede ou ISP.

  • Usuário - Isto é usado para fazer logon nos servidores de email.

  • Senha da conta - A senha deve ser forte, pois as contas de email muitas vezes estão disponíveis a partir de sites.

Os protocolos utilizados no email incluem os seguintes:

  • Post Office Protocol versão 3 (POP3)

  • Internet Message Access Protocol (IMAP)

  • Simple Mail Transfer Protocol (SMTP)

  • Multipurpose Internet Mail Extensions (MIME)

  • Secure Socket Layer (SSL)

Você precisa saber como configurar um dispositivo para aceitar o formato correto de emails recebidos. Você pode configurar o software cliente de email usando um assistente, como mostrado na Figura 2.

POP3

O POP3 recupera emails a partir de um servidor remoto via TCP/IP. O POP3 não deixar uma cópia do email no servidor, no entanto, algumas implementações permitem aos usuários especificar que o email é salvo por algum período de tempo. POP3 oferece suporte a usuários finais que possuem conexões intermitentes, como dial-up. Um usuário POP3 pode se conectar, baixar email a partir do servidor e, em seguida, se desconectar. POP3 geralmente usa a porta 110.

IMAP

O IMAP permite que os clientes de email locais recuperem emails de um servidor. Como o POP3, IMAP permite que você baixe email a partir de um servidor de email usando um software cliente. A diferença é que o IMAP permite ao usuário organizar os email no servidor de da rede e baixar cópias de email. O email original permanece no servidor da rede. Ao contrário do POP3, o IMAP normalmente deixa o e-mail original no servidor até que você mova o email em uma pasta pessoal em seu aplicativo. O IMAP sincroniza pastas de email entre o servidor e o cliente. O IMAP é mais rápido do que o POP3, porém requer mais espaço em disco no servidor e mais recursos da CPU. A versão mais recente do IMAP é IMAP4. IMAP4 é frequentemente utilizado em grandes redes, como um campus universitário. IMAP normalmente usa a porta 143.

SMTP

O SMTP é um protocolo simples, baseado em texto que transmite emails através de uma rede TCP/IP e é o formato de email para texto que usa apenas a codificação ASCII. SMTP deve ser implementado para enviar email. O SMTP envia mensagens de um cliente de email para um servidor ou de um servidor de para outro. Uma mensagem é enviada depois que os destinatários são identificados e verificados. O SMTP normalmente usa a porta 25.

MIME

O MIME estende o formato de email para incluir o texto no padrão ASCII, bem como outros formatos, como imagens e documentos de processamento de texto. O MIME normalmente usado em conjunto com o SMTP.

SSL

O SSL foi desenvolvido para transmitir arquivos de forma segura. Todos os dados trocados entre o cliente de email e o servidor são criptografados. Ao configurar um cliente de email para usar o SSL, certifique-se de usar o número correto da porta para o servidor de email.

Exchange

Exchange é um servidor de email, gerenciador de contatos e calendário criado pela Microsoft. O Exchange usa uma arquitetura de mensagens proprietária chamada Messaging Application Programming Interface (MAPI). O MAPI é usado pelo Microsoft Office Outlook para se conectar a servidores Exchange, fornecer email, calendário e gerenciamento de contatos.
