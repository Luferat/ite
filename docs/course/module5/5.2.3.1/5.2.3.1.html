Instalação do Sistema Operacional

Sequência de Inicialização e Arquivos de Registro

Compreender o processo de inicialização do Windows pode ajudar um técnico solucionar problemas de inicialização. A Figura 1 mostra a sequência de boot do Windows 7.

Processo de Inicialização do Windows

Para iniciar o processo de inicialização, ligue o computador. Isso é chamado de cold boot. Quando o computador é ligado, ele executa o Power On Self Test (POST). Devido ao fato de que a placa de vídeo ainda não foi inicializada, os erros que ocorrem neste momento no processo de boot são relatados por uma série de sinais sonoros, chamado códigos de beep.

Depois do POST, a BIOS localiza e lê as definições de configuração que são armazenados na memória CMOS. A prioridade de boot do dispositivo, tal como mostrado na Figura 2, é da ordem em que os dispositivos são verificados para localizar o sistema operacional. A prioridade de boot dos dispositivos é definido na BIOS e podem ser organizadas em qualquer ordem. A BIOS inicializa o computador usando o primeiro disco que contém um sistema operacional.

Os discos rígidos, unidades de rede, drives USB e mídias magnéticas removíveis, como cartões CompactFlash ou Secure Digital (SD), podem ser usados na ordem de inicialização, dependendo das capacidades da placa-mãe. Algumas BIOS também tem um menu de prioridade do dispositivo de inicialização, que é acessado com a combinação de teclas especiais, enquanto o computador é iniciado, antes da sequência de inicialização. Você pode usar este menu para selecionar o dispositivo de boot, que é útil se várias unidades podem inicializar o computador.

O Windows Boot Loader e do Gerenciador de Inicialização do Windows no Windows 7 e Windows Vista

Quando a unidade de armazenamento do S.O. é localizada, a BIOS encontra o Master Boot Record (MBR). Neste ponto, o Windows Boot Manager (BOOTMGR) controla várias etapas de instalação. Por exemplo, se houver mais de um sistema operacional presente no disco, o BOOTMGR dá ao usuário a chance de escolher qual usar. Se não houver outros sistemas operacionais, ou se o usuário não faz uma seleção antes do timer expirar, ocorre o seguinte processo:

1. O WinLoad usa o caminho especificado no BOOTMGR para encontrar a partição de boot.

2. O WinLoad carrega dois arquivos que compõem o núcleo do Windows 7: NTOSKRNL.EXE e HAL.DLL.

3. O WinLoad lê os Arquivos de Registro, escolhe um perfil de hardware e carrega os drivers de dispositivo.

NOTA: Se outra versão do S.O. Windows Vista ou posterior estiver instalada no disco, o BOOTMGR repetirá o processo. Se outra versão do S.O. Windows XP ou anterior estiver instalada no disco, o BOOTMGR invocará o Windows XP boot loader (NTLDR).

NTLDR e o Menu de Inicialização do Windows no Windows XP

Quando a unidade com o sistema operacional está localizada em um computador com o Windows XP, a BIOS localiza o MBR. O MBR localiza o boot loader NTLDR do sistema operacional. Neste ponto, o NTLDR controla várias etapas de instalação. Por exemplo, se houver mais de um sistema operacional presente no disco, o BOOT.INI dá ao usuário a chance de escolher qual usar. Se não houver outros sistemas operacionais, ou se o usuário não faz uma seleção antes do timer expirar, ocorre o seguinte processo:

1. O NTLDR executa o NTDETECT.COM para obter informações sobre o hardware instalado.

2. O NTLDR usa o caminho especificado no BOOT.INI para encontrar a partição de boot.

3. O NTLDR carrega dois arquivos que compõem o núcleo do XP: NTOSKRNL.EXE e HAL.DLL.

4. O NTLDR lê os Arquivos do Registro, escolhe um perfil de hardware e carrega os drivers de dispositivo.

NT Kernel

Neste ponto, o NT Kernel assume. O NT Kernel é o coração de todos os sistemas operacionais Windows. O nome desse arquivo é NTOSKRNL.EXE. Ele executa o arquivo de login chamado WINLOGON.EXE e exibe a tela de boas vindas do Windows.
