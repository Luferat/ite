Virtualização Client-Side

Propósito e Requisitos da Virtualização

Em um ambiente de negócios, as empresas devem gerenciar os recursos tecnológicos de uma forma que lhes permitam manter a competitividade, reduzindo custos e alocando os recursos com sabedoria. Como resultado, a virtualização client-side tornou-se um método popular de fornecer apoio à recursos críticos, tais como aplicações, serviços de compartilhamento de arquivos e outras ferramentas de produtividade. A virtualização também tem vantagens para usuários SOHO (Small Office/Home Office), pois pode fornecer acesso à programas que não estão disponíveis em um sistema operacional específico.

A virtualização ocorre quando uma máquina hospedeira (host) usa seus recursos de sistema para hospedar uma máquina virtual. Uma máquina virtual é também conhecida por guest. A máquina hospedeira deve ser uma máquina física, ligada e controlada por um usuário. A máquina virtual usa os recursos de sistema presentes na máquina física para inicializar e executar o sistema operacional. O sistema operacional da máquina virtual é independente do sistema operacional instalado na máquina host.

Hospedar de uma máquina virtual permite que os usuários acessem as funcionalidade e recursos fornecidos por um outro sistema operacional que está completamente isolado do sistema operacional do computador host. Por exemplo, uma máquina host executando o Windows 7 pode hospedar uma máquina virtual com o Windows XP instalado. Esta máquina virtual pode executar um software específico para o Windows XP. A instalação do Windows XP não interfere na instalação do Windows 7, presente na máquina host. Se necessário, os utilizadores podem aumentar ainda mais a funcionalidade dos seus recursos de sistema, executando múltiplas máquinas virtuais.
