Sequência de Inicialização e Arquivos de Registro

Registros do Windows

Os arquivos de registro do Windows são uma parte importante do processo de inicialização do sistema operacional. Esses arquivos são identificados pelos seus nomes específicos, que começam com HKEY_, conforme apresentado na figura, seguido pelo nome do local do sistema sob o seu controle. Cada configuração no Windows, desde a mudança do plano de fundo da área de trabalho, a cor dos botões da tela e até a licença das aplicações, são armazenadas nos Registros. Quando um usuário faz alterações nas configurações do Painel de Controle, associações dos arquivos, políticas do sistema ou instala novos softwares, essas alterações são armazenadas no Registro.

Cada conta de usuário possui uma seção dedicada nos Registros. Durante o processo de login do Windows, as configurações do sistema presentes no Registro são acessadas para reconfigurar o sistema para cada conta de usuário.

Os Registros também são responsáveis por gravar a localização dos arquivos DLL (Dynamic Link Library). Um arquivo DLL consiste em um código de programa que pode ser utilizado por diferentes aplicações para executar as tarefas comuns. Como resultado, os arquivos DLL são muito importantes para a funcionalidade de um sistema operacional e para todos os programas que os usuários venham a instalar.

Para garantir que uma DLL possa ser localizada pelo sistema operacional ou por um programa, ela deve ser registrada. Normalmente, uma DLL é registrada automaticamente durante o processo de instalação. Pode ser necessário que um usuário tenha que registrar manualmente uma DLL quando um problema for identificado. Registrar uma DLL mapeia o caminho para acessar o arquivo, tornando mais fácil para os programas localizar as DLLs necessárias. Para registrar um arquivo DLL no Windows usando o prompt de comando, use o seguinte caminho:

Iniciar > digite cmd > digite regsvr32 Nome_Do_Arquivo.dll
